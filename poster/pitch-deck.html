The Problem

The data exists. It's just scattered everywhere.

🗂️

Fragmented across systems

Evaluation scores live in one portal. Bidding records in another. Program requirements in a third. Schedules in a fourth. Students must visit 4+ tools — and reconcile the results themselves.

→ Synthesis burden falls on the student
💸

Bidding without context

Raw bidding data exists — but without historical clearing prices, capacity trends, or peer benchmarks, students can't act on it. Overbidding wastes points; underbidding loses seats.

→ Systematic resource misallocation
📋

No unified planning layer

Program offices and RAs repeatedly answer the same questions: what counts toward requirements, which sections conflict, which professor has the better reviews. No self-service tool exists.

→ Advisor time burned on solvable queries
"The data was always there — evaluations, bidding, requirements — just never in one place where you could actually think with it." — MPP student, HKS Class of 2025 (composite)

The Solution

One tool. All the signal.

HKS Course Explorer pulls together data that already exists across HKS systems — evaluations, bidding records, program requirements, scheduling — into one fast, filterable, mobile-ready app for students and program offices alike.
